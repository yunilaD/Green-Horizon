Home Page and Global CSS

Technical Description

The Home page was designed to serve as an engaging landing page that immediately communicates the site's purpose and mission. The HTML structure follows semantic principles with a clear document hierarchy, utilizing the following key elements:

The page begins with a responsive header that includes our logo and navigation menu, implemented using unordered lists for accessibility. The hero section features a large background image with overlay text to create visual impact, implemented with div elements and CSS positioning.

Key CSS techniques used include:

  1. Flexbox layout for navigation bar and content tiles (display: flex) for fluid positioning
  2. CSS positioning (position: relative/absolute) for overlay text and elements
  3. Box-sizing: border-box to maintain consistent element dimensions
  4. Media queries for responsive design across different screen sizes
  5. CSS transitions for hover effects on navigation links and buttons
  6. Icon integration using Boxicons (bx icons class) for visual enhancement

The global CSS establishes consistent styling across the website with a color scheme dominated by green tones that reinforce the environmental theme. Typography uses the Poppins font family for readability, while the responsive footer is structured with flexbox for optimal layout across devices.

Accessibility

Several measures were implemented to ensure accessibility:

  1. Semantic HTML5 tags for improved screen reader navigation
  2. Alt text for all images, including the logo and initiative thumbnails
  3. Sufficient color contrast between text and backgrounds (green text on white, white text on dark backgrounds)
  4. Responsive design that works across various screen sizes and devices
  5. Navigation links with clear hover states for better user feedback
  6. Consistent heading hierarchy (h1, h2, h3) throughout the page
  7. Focus states for interactive elements to aid keyboard navigation

Table

Technical Description

The Endangered Animals table page was designed to present structured data about endangered species in an interactive and informative format. The page utilizes the following key HTML and CSS techniques:

The HTML structure employs semantic table elements (table, thead, tbody, tr, th, td) to organize data in a logical way. Data attributes (data-status, data-region) were added to table rows to enable filtering functionality. The table includes sortable columns, search functionality, and filtering options.

Key CSS techniques used include:

  1. Custom table styling with border-collapse for clean grid lines
  2. Status indicators using colored spans with distinct classes (.critical, .endangered, .vulnerable)
  3. Interactive elements including search input, filter dropdowns, and detail buttons
  4. Responsive table container with overflow handling for smaller screens
  5. Boxicons integration for sort indicators and status icons
  6. Stat cards using flexbox for the Conservation Status Overview section
  7. Consistent color coding (red for critical, orange for endangered, yellow for vulnerable) to visually communicate status

Accessibility

The table page incorporates numerous accessibility features:

  1. Proper table structure with thead and tbody for screen reader navigation
  2. Descriptive table headers for each column
  3. Color is not the only indicator of status - text labels accompany each color
  4. Search and filter controls are clearly labeled
  5. Button elements used for interactive controls rather than styled divs
  6. Sufficient color contrast for all text elements
  7. Visual indicators (icons) paired with text for dual communication methods
  8. Responsive design that maintains readability on different screen sizes

Challenges and Lessons Learned

During the implementation of these pages, several challenges were encountered:

  1. Responsive Design: Creating a layout that worked seamlessly across different screen sizes required multiple iterations. I learned to use flexible units (%, vh/vw) instead of fixed pixels, and implemented media queries to adjust layouts for mobile devices.
  2. Table Interactivity: Implementing the filtering and sorting functionality for the endangered animals table presented challenges. I developed a better understanding of DOM manipulation techniques and data attributes.
  3. CSS Organization: Managing global styles versus page-specific styles required careful planning. I learned to structure CSS files more efficiently, using descriptive class names and avoiding overly specific selectors.
  4. Cross-browser Compatibility: Ensuring consistent appearance across different browsers required testing and adjustments. I discovered the importance of using standardized properties and vendor prefixes where necessary.
  5. Performance Optimization: Large background images affected page load times. I learned techniques for image optimization and proper sizing to balance visual quality with performance.

These challenges taught me the importance of planning before coding, testing across different devices, and implementing progressive enhancement to ensure basic functionality works for all users while providing enhanced experiences for those with modern browsers.

Compliance

The pages developed comply with Janet regulations for web page publication in the following ways:

  1. Accessibility Standards: Pages implement WCAG guidelines with semantic HTML, alt text for images, and keyboard navigation support.
  2. Security Practices: No user data collection occurs without proper notification, and all links to external resources use secure connections where appropriate.
  3. Content Appropriateness: All content is educational and informative, with no inappropriate material. Sources are cited for factual information.
  4. Responsive Design: Pages adapt to different screen sizes, ensuring accessibility across devices as required by modern web standards.
  5. Performance Considerations: Images are optimized to reduce page load times, improving user experience and reducing bandwidth usage.
  6. Branding Guidelines: The Green Horizon branding is consistently applied throughout all pages, maintaining a professional appearance.

The development process prioritized creating clean, standards-compliant code that meets both technical and regulatory requirements while delivering an engaging user experience.
