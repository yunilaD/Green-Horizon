Volunteer Page

Technical Description

The Volunteer page is designed to provide users with structured and accessible information about different volunteering opportunities related to environmental conservation. The page is developed using HTML5 for structure, CSS3 for styling, and JavaScript for interactivity. It begins with a hero section, which features a heading and a brief introductory paragraph to engage users. Below this, a sorting filter allows users to arrange volunteer opportunities based on popularity, duration, or recency.

The core of the page consists of volunteer cards, each representing a different program. These cards include an image, a title, a short description, and key details such as duration, location, and skill level. The page also contains a ratings and reviews section, where users can read feedback from previous volunteers. A dedicated feedback section enables users to submit reviews using emoji-based ratings and a text box for comments. The overall design is clean, intuitive, and structured to enhance user engagement.

Accessibility

To ensure accessibility, the Volunteer page follows best practices in semantic HTML, providing a clear document structure with elements like <header>, <main>, <section>, and <footer>. All images include alt attributes to make the content accessible to screen readers. The color scheme is chosen for high contrast, ensuring readability for visually impaired users. The sorting dropdown and buttons are keyboard accessible, allowing navigation using the Tab key.

Additionally, interactive elements such as buttons and form fields are properly labeled to enhance usability with assistive technologies. The use of structured headings (h1, h2, h3) improves screen reader navigation, making the content more comprehensible.

Splash Screen

Technical Description

The Splash Screen serves as an introductory page that briefly welcomes users before redirecting them to the homepage. The page features a full-screen video background, implemented using the <video> element, with an overlay to enhance text readability. The page includes a logo section, a mission statement, and a team members section to provide context about the project.

The key functionality is the automatic redirection to the home page after 4 seconds, achieved using the <meta http-equiv="refresh" content="4; url=home.html"> tag. Additionally, a loading animation with a progress bar visually represents the time remaining before redirection. JavaScript is used to smoothly animate the progress bar from 0% to 100%, creating a seamless transition for users.

Accessibility

The Splash Screen is designed to be inclusive and user-friendly. A fallback message is provided for users whose browsers do not support videos, ensuring they still receive the necessary information. The text content is positioned over an overlay, which enhances contrast and readability. The progress bar animation is visually clear, providing feedback about the redirection process.

Although the page has minimal interactive elements, it follows keyboard accessibility guidelines, ensuring that users who rely on assistive technologies can access the content before the automatic transition. Additionally, the font selection and text size have been chosen for optimal legibility.

Content Page

Technical Description

The wildlife conservation content page is designed to align with the home and table pages, ensuring a consistent user experience. It features a header and footer for seamless navigation, clearly structured content with a logical heading hierarchy, and responsive media elements such as images and videos with descriptive alt text. Interactive elements are incorporated where appropriate to enhance user engagement. The page maintains a clean, professional look with the Poppins font and the established green color scheme, reinforcing brand identity. Structured for clarity and accessibility, it utilizes semantic HTML5 elements like <section>, <article>, and <aside< to present information in an organized and intuitive way.

Accessibility

The content page will incorporate essential accessibility features to ensure an inclusive user experience. It will use a semantic HTML structure for screen reader compatibility and provide descriptive alt text for all images. Interactive elements will be fully navigable by keyboard, with ARIA labels added where necessary to support complex interactions. A consistent heading structure will maintain a clear document outline, and all text content will meet sufficient color contrast standards for readability. Additionally, the page will feature a responsive design, adapting seamlessly to different screen sizes and devices.

Challenges and Lessons Learned

One of the biggest challenges with the Volunteer page was making sure it looked good and worked smoothly on all screen sizes. Getting the volunteer cards to adjust properly on mobile without breaking the layout took some tweaking with CSS. Another tricky part was the sorting feature—figuring out how to let users filter opportunities easily while keeping things simple and user-friendly.

Ensuring accessibility was also a priority, making sure screen readers could navigate the page properly and that all images had alt text. The main issue with the video background was its large file size, slowing down the page, especially on mobile devices. To fix this, we had to optimize the video while keeping good quality. The automatic redirection was another challenge—balancing a smooth transition without making it too fast for screen readers to process.

Through this, we learned the importance of responsive design, accessibility, and performance optimization. Mobile-first design made everything easier, and small tweaks like lazy loading images and compressing videos helped speed up the site. Most importantly, testing with real users helped us see what worked and what didn’t. In the end, it was a great learning experience in building efficient, user-friendly web pages.

Compliance

Ensuring that the Volunteer and Splash Screen pages meet web standards and accessibility guidelines was a key focus during development. Both pages follow HTML5 and CSS3 best practices, using semantic elements like "header," "main," and "footer" to create a well-structured and readable document.

For accessibility compliance, we considered WCAG (Web Content Accessibility Guidelines) by adding alt text for images, maintaining high color contrast, and ensuring that all interactive elements are keyboard accessible. The Splash Screen’s automatic redirection was kept at a reasonable time to avoid interfering with screen readers.

We also followed performance best practices by optimizing media files and using efficient CSS for better loading times. Validation tools like W3C HTML & CSS validators were used to check for errors and ensure clean, compliant code. These efforts help ensure the pages are inclusive, functional, and user-friendly across different devices and browsers.
